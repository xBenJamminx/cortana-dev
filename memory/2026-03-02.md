## Session Persistence Fix -- March 2 ~06:40 UTC

**Problem:** Ben kept losing conversation context between messages. Every gap of >60 minutes meant Cortana started a brand new session with no memory of prior conversation.

**Root cause found:** OpenClaw's default session config:
- Reset mode: "daily" (wiped all sessions at 4am)
- Idle timeout: 60 minutes (any gap >1hr = new session)
- Both applied to ALL session types including topic threads

**Fix applied:** Edited `/root/.openclaw/openclaw.json`, added `session` config block:
- Reset mode changed: "daily" → "idle" (no more 4am wipe)
- Idle timeout changed: 60min → 1440min (24 hours)
- Applied to all types: direct, group, thread/topic via `resetByType`

**Technical details:**
- Session key for topic 31: `agent:main:telegram:group:-1003856131939:topic:31`
- Freshness check in `sessions-DOpDuDMA.js` line 327: `evaluateSessionFreshness()`
- Reset type for topics: "thread" (detected by `:topic:` in session key)
- Claude CLI resume works via `--resume {sessionId}` when session is "fresh"
- When session is stale, new UUID is generated and old context is lost

**Verification:** Config valid, gateway auto-restarted, session config confirmed present.
