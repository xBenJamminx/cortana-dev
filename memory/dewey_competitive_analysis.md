## 7. TECHNICAL APPROACH

### How It Works
- **Primary method:** Chrome extension scraping (NOT API-dependent)
- Extension adds "Grab Bookmarks" button on Twitter's bookmarks page
- Clicks trigger the extension to pull in bookmarks from the Twitter/X interface directly
- This bypasses the Twitter API's 800 bookmark limit
- For platforms like Bluesky (no bookmark API), extension-based scraping is the only method
- Users must be on the bookmarks page and logged in for it to work

### API vs Scraping
- Dewey claims they don't rely on APIs in "99.999% of cases"
- They ask users to use the Chrome extension instead of API sync
- API serves as a fallback but is unreliable (especially X's API)
- Extension uses OAuth authorization through Twitter's flow

### Permissions Required
- Storage, scripting, webRequest, activeTab
- Host permissions: getdewey.co, multiple social media platforms (Bluesky, X/Twitter, LinkedIn, Threads, etc.)
- Access to browser tabs

### Known Technical Limitations
- **800 bookmark API limit** (when using API fallback instead of extension)
- Chrome-only (no Firefox, Safari, or mobile)
- Requires being on the target platform's page to grab bookmarks
- Manual sync on free plan (must click button each time)
- Dark mode CSS conflicts (white text on white background)
- Bookmark grab process can stall, hang, or silently fail
- Duplicate imports possible if process runs twice

## 10. WHERE DEWEY FAILS (OPPORTUNITY MAP)

### Critical Failures

1. **Reliability is the #1 problem.** The bookmark grabbing process frequently fails silently, hangs, logs users out, or imports incomplete sets. This is the core value proposition and it doesn't work reliably.

2. **Cross-platform promises are hollow.** They market 10+ platforms but users report TikTok, Instagram, and other integrations are broken or non-functional. The multi-platform story is aspirational, not real.

3. **Support is non-responsive.** Days to respond, no solutions provided. For a $10/mo tool, this is unacceptable. 2-person team can't keep up.

4. **Chrome-only.** No Firefox, Safari, mobile app. They promised a mobile app in 2025 (per GrabLTD listing) -- appears not delivered.

5. **Paywall friction.** Free plan is manual-only with 1 account. The auto-sync that makes the tool actually useful is locked behind $10/mo. Export Pass ($50 for 48 hours) is predatory pricing.

6. **Dark mode breaks the extension.** White text on white background. Basic CSS issue that persists.

7. **Security permissions are excessive.** Requests access that makes security-conscious users uncomfortable. "Very high risk impact" rating from chrome-stats.

8. **No real AI intelligence.** AI tagging is keyword-based, not semantic. No content understanding, no smart recommendations, no insights from bookmark patterns.

9. **Duplicate handling is poor.** Running grab twice creates duplicates with no dedup logic.

10. **1,272 Twitter followers** for a Twitter bookmark manager. Their own social media presence is weak, suggesting low brand loyalty and engagement.

### Structural Weaknesses

- **2-person team** trying to support 10+ platform integrations = stretched impossibly thin
- **$149K revenue** in 2023 with 45K users = low conversion (~3.3% paid)
- **Lifetime deals** ($129-$225) signal revenue desperation and devalue the subscription
- **Blog is SEO-heavy but low-quality** -- many posts are generic, AI-generated content farming
- **Rating decline** from initial ~4.0 to current 1.8-3.5 range shows product degradation over time

### What Users Actually Want (Based on Complaints)

1. **Reliable syncing that just works** -- no manual clicking, no failures, no incomplete imports
2. **Real cross-platform support** -- not just Twitter but TikTok, LinkedIn, Instagram that actually functions
3. **Fast, responsive support** -- or better yet, a product that doesn't need support
4. **Mobile access** -- bookmarks are consumed on mobile, managed on desktop is a poor UX
5. **Smart organization** -- not keyword matching but actual content understanding
6. **Data portability** -- easy export without a $50 "Export Pass" grift
7. **Privacy-respecting permissions** -- minimal footprint, transparent data handling
